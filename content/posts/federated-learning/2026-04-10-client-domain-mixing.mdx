## 왜 기록해야 하나

Federated Learning 실험은 같은 모델과 같은 seed를 사용해도 client sampling과 domain 구성이 바뀌면
결과 해석이 달라집니다. 따라서 loss만 저장하면 실패 원인을 찾기 어렵습니다.

## 최소 기록 단위

round마다 다음 정보를 남기는 것을 기본값으로 잡았습니다.

| Field | Description |
| --- | --- |
| round | server round |
| selected_clients | 선택된 client id |
| domain | client가 대표하는 domain |
| num_examples | local sample 수 |
| train_loss | local train loss |
| eval_metric | 공통 평가 metric |

## 구현 기준

client 함수가 metric dict를 반환하고, server는 이를 그대로 JSONL로 저장합니다.

```python
metrics = {
    "client_id": client_id,
    "domain": domain,
    "num_examples": len(train_dataset),
    "train_loss": float(loss),
}
```

## 정리

실험 로그는 나중에 결과를 해석하기 위한 데이터셋입니다. 모델 성능표보다 먼저 신뢰할 수 있는 로그 구조를
만드는 편이 장기적으로 시간을 줄여줍니다.
